# Design Skill - Ground Crew HQ

## Purpose
This skill defines when and how to apply the Ground Crew HQ visual design system.

`docs/design.md` is the source of truth for visual design decisions, layout polish, branding consistency, and UI quality direction.

## When To Use This Skill
Use `design-skill` for:
- UI and visual polish
- Layout refinement
- Branding alignment
- Dashboard presentation quality
- Modal/form experience polish
- Component consistency and hierarchy

Applies across:
- app shell
- dashboard
- employees
- scheduler
- workboard
- equipment
- weather
- reports
- settings
- landing page

## When NOT To Use This Skill
Do not use `design-skill` for:
- auth-only tasks
- SQL-only tasks
- backend-only tasks
- bug-only tasks with no UI impact

If a bug includes UI behavior or visual state handling, then this skill is relevant.

## Codex Instruction
Read `docs/design.md` before making visual changes.

## Implementation Rule
Make small page-specific improvements. Do not redesign the entire app in one pass.
